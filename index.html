Tekken 7 Modding

A hex-editor is used to modify the character's moveset/fighting moves. The first challenge was the finding the memory address of where the player's current move is stored. Once the address was found, the values stored in the address can now be seen and editied (the values must be converted to hexadecimals or decimals to be readable). Changing the values within the address will cause the player's character to go into different states or do different moves, thus using guess and check is the method I used to find the possible moves/states that are possible in the character. However, inputing an invalid number into the address will cause the game to crash which can delay the process of modifying the game. I found moves that are not meant for players to access and record the gameplay these moves to post it on YouTube. Macros are used to access these hidden moves.

Pacman

The Pacman project is the first project I have done for my University assignment. The game has all the basic components of a Pacman game such as collisions, animations, sound effects, scoreboards, powerups and enemy AI. A tile map system is used to load in the level, where the program reads from a text file to get the coordinates of the walls, munchies, ghosts, etc... and renders them in the correct position. For the enemy AI, each colored ghosts have a different AI. The orange ghosts goes either in the verticle or horizontal direction and changes direction when it hits the wall, the blue ghosts randomly changes direction when it hits the wall, the red ghosts is programmed to patrol in a square box pattern or motion and the pink ghosts constantly chases Pacman with the ability to respawn and pass through walls.

Mario

For my Mario Project, SDL2 library was used. There are two levels available currently, with three different types of enemies. I managed to create a Screen Manager to switch between the Main Menu to the different levels. Sound effects and music is also present which is done using the Mix_Chunk struct and Mix_PlayChannel function. The player has to kill 8 Koopas to clear a level and enemies spawn from the pipes. I also tried to add new features such as the yellow platform, where the player can jump through it and a Green POW block that gives the play speed boost for six seconds.

3D Room

For my 3D Room Project, the OpenGl library was used to create the Scene. My project has a free moving camera which the user can control using WASD to move around, with being able to rotate. The scene contains objects rendered from .obj that is created in Blender and text files. Textures are loaded with .tga and .raw files to texture the objects. The objects and light source can be moved using user input in any coordinates.

Sword Cube

The initial plan for Sword Cube is to make a story driven game where the player goes through the forest and fight through enemies to save his brother, but the game direction changed to a wave based melee combat survivor game where the player has to survive as many waves as possible. There are 3 types of slimes, normal slimes (blue), fast slimes (green) and tanky slimes (brown). The player and the slimes get stronger each wave, the player can choose between 4 power ups, red (more health), blue (more block stamina), green (more movement speed) and brown (more attack damage). The game has simple controls where the player can attack, block and heal which can be made into a mobile game

The Spirit's Tale

The Spirit's Tale is a group project made by seven people where we split our duties to finish this game. The is a platformer, stealth and a puzzle game where the player has to jump through platforms, avoid enemies and solve puzzles to beat levels. There are puzzle elements such as pushable/throwable boxes which can trigger pressure plates, levers to pull to open doors and light receivers that activates/deactivates puzzle elements depending on whether light is shining on it or not. In this project, I helped creating the enemy AI and made some of the levels with puzzles for the player to solve. A State Machine is created to determine the state of the black-cloacked enemy (Patroling, Suspicious, Chase and Return to Patrol). When the enemy is in the patroling state where they move from one point to another. If the enemy sees the player for a certain amount of time, the enemy will chase the player. A-Star Pathfinding is used to chase the player around with efficiency. To make this balanced, the enemy moves slightly slower than the player and it takes time for them to do jumps.

Desert Cube

Desert Cube is a 2D-Rogue-Lite game that I made during my 2nd Year at University. The goal of the player is to survive as many waves as possible. As the waves progress, enemies will be stronger (more damage, hp) and more of them will appear each wave. Between each wave, the player can spend the gold that they earned to increase the strength of the player's character. The player can also choose to buy challenges, a Risk and Reward system where the player will be in a handicap for the next wave, but rewarded extra gold if the player survives the game. There different types of enemies, that have special attributes such as increased speed, hp, or damage which drops extra gold when killed. The player can Chain Attacks, Block, Parry, Use Special Abilities (Bought from the Shop) and Dodge Roll

FPTrainer

FPTrainer is a local multiplayer FirstPersonShooter, where 2 players duel one another (deathmatch mode where players respawn after death) until one player reaches the winning score. Each player have 3 different guns with each having their unique strengths. Jump pads are present in the game to introduce exciting moments such as trick-shots and increase mobility movement around the map.

Cube Defence

Cube Defence is a tower defence game but with a controllable player character in the game. The player can move around different blue vantage points to get in range to shoot at enemies by acting similarly to a tower. Mr. Red will move through his vantage points to attack the player's tower and base, he must be stopped to prevent further damage.

PlatJumper

PlatJumper is an endless-runner mobile game, where the goal is to get as high as possible by pressing the correct arrow-directions to jump up the platforms. Mr. Red is the antagonist of the game where he will push the player off the platform if he reaches them.
